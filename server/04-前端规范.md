## 2. 全局 Provider 与基础布局

`src/app/layout.tsx`：

```tsx
import type { ReactNode } from 'react'
import { ConfigProvider, App as AntdApp } from 'antd'
import zhCN from 'antd/locale/zh_CN'
import 'antd/dist/reset.css'
import { TRPCProvider } from '@/lib/trpc-client'

export const metadata = { title: '库存管理系统' }

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="zh-CN">
      <body>
        <TRPCProvider>
          <ConfigProvider locale={zhCN}>
            <AntdApp>{children}</AntdApp>
          </ConfigProvider>
        </TRPCProvider>
      </body>
    </html>
  )
}
```

> **Toast**：使用 `<AntdApp>` 包裹后，组件内部用 `App.useApp().message` 弹 toast。**所有成功/失败提示统一通过它**，不要自己 `message.xxx`，否则文案不会被语义识别为 `role="alert"`。
> **Antd `<App>` 的 message 默认会渲染带 `role="alert"` 容器**；test-agent 用 `getByRole('alert')` 即可。

`src/app/(app)/layout.tsx`（带 sidebar+header）：

```tsx
'use client'
import { Layout, Menu, Spin } from 'antd'
import Link from 'next/link'
import { useRouter } from 'next/navigation'
import { useEffect } from 'react'
import { trpc } from '@/lib/trpc-client'

export default function AppLayout({ children }: { children: React.ReactNode }) {
  const router = useRouter()
  const me = trpc.auth.me.useQuery(undefined, { staleTime: 60_000 })
  useEffect(() => {
    if (me.isFetched && me.data === null) router.replace('/login')
  }, [me.isFetched, me.data, router])
  if (!me.isFetched || me.data === null) return <Spin />

  return (
    <Layout style={{ minHeight: '100vh' }}>
      <Layout.Sider>
        <Menu
          theme="dark"
          mode="inline"
          items={[
            { key: 'dashboard', label: <Link href="/dashboard">仪表板</Link>, ['data-testid' as never]: 'nav-dashboard' },
            { key: 'products', label: <Link href="/products">商品管理</Link>, ['data-testid' as never]: 'nav-products' },
            { key: 'categories', label: <Link href="/categories">分类管理</Link>, ['data-testid' as never]: 'nav-categories' },
            { key: 'suppliers', label: <Link href="/suppliers">供应商管理</Link>, ['data-testid' as never]: 'nav-suppliers' },
            { key: 'purchase-orders', label: <Link href="/purchase-orders">进货管理</Link>, ['data-testid' as never]: 'nav-purchase-orders' },
            { key: 'sales-orders', label: <Link href="/sales-orders">出货管理</Link>, ['data-testid' as never]: 'nav-sales-orders' },
            { key: 'stats', label: '库存统计', children: [
              { key: 'stats-alerts', label: <Link href="/stats/alerts">库存预警</Link>, ['data-testid' as never]: 'nav-stats-alerts' },
              { key: 'stats-trends', label: <Link href="/stats/trends">库存趋势</Link>, ['data-testid' as never]: 'nav-stats-trends' },
              { key: 'stats-report', label: <Link href="/stats/report">库存报表</Link>, ['data-testid' as never]: 'nav-stats-report' },
            ]},
          ]}
        />
      </Layout.Sider>
      <Layout>
        <Layout.Header style={{ background: '#fff', display: 'flex', justifyContent: 'flex-end', gap: 12, alignItems: 'center' }}>
          <span>{me.data.username}</span>
          <LogoutButton />
        </Layout.Header>
        <Layout.Content style={{ padding: 24 }}>{children}</Layout.Content>
      </Layout>
    </Layout>
  )
}

function LogoutButton() {
  const router = useRouter()
  const utils = trpc.useUtils()
  const logout = trpc.auth.logout.useMutation({
    onSuccess: async () => {
      await utils.auth.me.reset()
      router.replace('/login')
    },
  })
  return (
    <button data-testid="header-logout" onClick={() => logout.mutate()}>
      退出登录
    </button>
  )
}
```

> **未登录访问 `(app)` 任意页 → 立即客户端跳 `/login?redirect=<原路径>`**（测试要验证）。
> 服务端鉴权：每个 protected procedure 已经判断 `ctx.user`，即便客户端跳转失败也安全。

## 3. 页面骨架模板（每模块一致）

### 3.1 列表页骨架（举例：products）

```tsx
'use client'
import { Table, Button, Input, Select, Space, Modal, App as AntdApp } from 'antd'
import Link from 'next/link'
import { useState } from 'react'
import { trpc } from '@/lib/trpc-client'

export default function ProductsPage() {
  const { message } = AntdApp.useApp()
  const [keyword, setKeyword] = useState('')
  const [categoryId, setCategoryId] = useState<number | undefined>()
  const [stockStatus, setStockStatus] = useState<'ALL'|'LOW'|'OVER'|'NORMAL'>('ALL')
  const [page, setPage] = useState(1)
  const [pageSize, setPageSize] = useState(20)
  const [selected, setSelected] = useState<number[]>([])

  const list = trpc.products.list.useQuery({ keyword, categoryId, stockStatus, page, pageSize })
  const cats = trpc.categories.list.useQuery({})
  const utils = trpc.useUtils()

  const del = trpc.products.delete.useMutation({
    onSuccess: () => { message.success('删除成功'); utils.products.list.invalidate() },
    onError: (e) => message.error(translateError(e.message)),
  })
  const batchDel = trpc.products.batchDelete.useMutation({
    onSuccess: (r) => {
      message.success('删除成功')
      if (r.failed.length) message.error(`部分失败：${r.failed.length} 项`)
      utils.products.list.invalidate()
    },
  })

  return (
    <div>
      <Space style={{ marginBottom: 16 }}>
        <Input
          placeholder="搜索编码或名称"
          aria-label="搜索"
          data-testid="products-search"
          value={keyword}
          onChange={(e) => { setKeyword(e.target.value); setPage(1) }}
        />
        <Select
          placeholder="选择分类"
          aria-label="分类筛选"
          allowClear
          style={{ minWidth: 160 }}
          options={cats.data?.map((c) => ({ label: c.name, value: c.id })) ?? []}
          value={categoryId}
          onChange={(v) => { setCategoryId(v); setPage(1) }}
        />
        <Select
          aria-label="库存状态"
          options={[
            { label: '全部', value: 'ALL' },
            { label: '低库存', value: 'LOW' },
            { label: '超容量', value: 'OVER' },
            { label: '正常', value: 'NORMAL' },
          ]}
          value={stockStatus}
          onChange={(v) => { setStockStatus(v); setPage(1) }}
        />
        <Link href="/products/new">
          <Button type="primary" data-testid="products-create">新增商品</Button>
        </Link>
        <Button
          data-testid="products-batch-delete"
          danger
          disabled={selected.length === 0}
          onClick={() =>
            Modal.confirm({
              title: '确认批量删除？',
              content: `共 ${selected.length} 项，删除后不可恢复，是否继续？`,
              okText: '确认',
              cancelText: '取消',
              okButtonProps: { 'data-testid': 'modal-confirm' as never },
              cancelButtonProps: { 'data-testid': 'modal-cancel' as never },
              onOk: () => batchDel.mutate({ ids: selected }),
            })
          }
        >
          批量删除
        </Button>
      </Space>

      <Table
        data-testid="products-list"
        rowKey="id"
        rowSelection={{
          selectedRowKeys: selected,
          onChange: (keys) => setSelected(keys as number[]),
        }}
        loading={list.isLoading}
        dataSource={list.data?.items ?? []}
        pagination={{
          current: page,
          pageSize,
          total: list.data?.total ?? 0,
          onChange: (p, ps) => { setPage(p); setPageSize(ps) },
          itemRender: (_p, type, original) => {
            if (type === 'prev') return <a data-testid="pagination-prev">{original}</a>
            if (type === 'next') return <a data-testid="pagination-next">{original}</a>
            if (type === 'page') return <a data-testid={`pagination-page-${_p}`}>{original}</a>
            return original
          },
        }}
        columns={[
          { title: '编码', dataIndex: 'code', key: 'code' },
          { title: '名称', dataIndex: 'name', key: 'name' },
          { title: '分类', key: 'category', render: (_v, r) => r.category.name },
          { title: '当前库存', dataIndex: 'quantity', key: 'quantity' },
          { title: '单位', dataIndex: 'unit', key: 'unit' },
          { title: '进价', dataIndex: 'costPrice', key: 'costPrice' },
          { title: '售价', dataIndex: 'sellPrice', key: 'sellPrice' },
          { title: '供应商', key: 'supplier', render: (_v, r) => r.supplier.name },
          {
            title: '操作',
            key: 'action',
            render: (_v, r) => (
              <Space>
                <Link href={`/products/${r.id}/edit`}>
                  <a data-testid={`row-${r.id}-edit`}>编辑</a>
                </Link>
                <a
                  data-testid={`row-${r.id}-delete`}
                  onClick={() =>
                    Modal.confirm({
                      title: '确认删除？',
                      content: '删除后不可恢复，是否继续？',
                      okText: '确认',
                      cancelText: '取消',
                      okButtonProps: { 'data-testid': 'modal-confirm' as never },
                      cancelButtonProps: { 'data-testid': 'modal-cancel' as never },
                      onOk: () => del.mutate({ id: r.id }),
                    })
                  }
                >
                  删除
                </a>
              </Space>
            ),
          },
        ]}
        onRow={(r) => ({ 'data-testid': `row-${r.id}` })}
      />
    </div>
  )
}

function translateError(msg: string): string {
  // 完整映射见 00-技术约束.md §7.2；此处省略。
  const map: Record<string, string> = {
    PRODUCT_HAS_STOCK: '库存量大于0的商品不能删除，请先清空库存',
    PRODUCT_NOT_FOUND: '商品不存在',
  }
  return map[msg] ?? '操作失败'
}
```

> 上面是骨架；frontend-impl-agent 必须**逐字落地** `data-testid`、`aria-label`、错误文案映射。

### 3.2 表单页骨架（举例：products/new）

复用 `<ProductForm>` 组件（见 `demo/src/components/products/product-form.tsx`，但需要把 antd Form 的 `data-testid` 与 label 对齐）：

```tsx
// src/components/products/product-form.tsx
'use client'
import { Form, Input, InputNumber, Select, Button } from 'antd'
import { useState } from 'react'
import { z } from 'zod'

export type ProductFormValues = {
  code: string
  name: string
  categoryId: number
  description?: string
  costPrice: number
  sellPrice: number
  quantity: number
  unit: string
  supplierId: number
  minStock: number
  maxStock: number
}

const schema = z
  .object({
    code: z.string().regex(/^[A-Za-z0-9-]{3,20}$/, '编码必须为3-20位字母数字或短横线'),
    name: z.string().trim().min(1, '名称不能为空').max(100, '名称长度不能超过100'),
    categoryId: z.number().int().positive('请选择分类'),
    description: z.string().max(1000, '描述长度不能超过1000').optional(),
    costPrice: z.number().positive('进价必须大于0').max(99999.99)
      .refine((n) => Math.round(n * 100) === n * 100, '进价最多保留2位小数'),
    sellPrice: z.number().positive('售价必须大于0').max(99999.99)
      .refine((n) => Math.round(n * 100) === n * 100, '售价最多保留2位小数'),
    quantity: z.number().int().nonnegative('库存数量必须 >= 0'),
    unit: z.string().min(1, '请填写单位').max(20),
    supplierId: z.number().int().positive('请选择供应商'),
    minStock: z.number().int().positive('最小库存必须 > 0'),
    maxStock: z.number().int(),
  })
  .refine((d) => d.maxStock > d.minStock, {
    message: '最大库存必须大于最小库存',
    path: ['maxStock'],
  })

export function ProductForm({
  initial,
  categories,
  suppliers,
  onSubmit,
  mode = 'create',
}: {
  initial?: Partial<ProductFormValues>
  categories: { id: number; name: string }[]
  suppliers: { id: number; name: string }[]
  onSubmit: (values: ProductFormValues) => Promise<void>
  mode?: 'create' | 'edit'
}) {
  const [form] = Form.useForm()
  const [submitting, setSubmitting] = useState(false)
  const [errors, setErrors] = useState<Record<string, string>>({})

  async function handleFinish(raw: any) {
    setErrors({})
    const parsed = schema.safeParse({
      ...raw,
      categoryId: Number(raw.categoryId ?? 0),
      supplierId: Number(raw.supplierId ?? 0),
      costPrice: Number(raw.costPrice ?? 0),
      sellPrice: Number(raw.sellPrice ?? 0),
      quantity: Number(raw.quantity ?? 0),
      minStock: Number(raw.minStock ?? 0),
      maxStock: Number(raw.maxStock ?? 0),
    })
    if (!parsed.success) {
      const e: Record<string, string> = {}
      for (const issue of parsed.error.issues) {
        const k = String(issue.path[0])
        if (!e[k]) e[k] = issue.message
      }
      setErrors(e)
      return
    }
    setSubmitting(true)
    try { await onSubmit(parsed.data) } finally { setSubmitting(false) }
  }

  return (
    <Form
      form={form}
      layout="vertical"
      initialValues={initial}
      onFinish={handleFinish}
      aria-label="商品表单"
    >
      <Form.Item label="编码" name="code">
        <Input disabled={mode === 'edit'} aria-label="编码" />
      </Form.Item>
      {errors.code && <p role="alert">{errors.code}</p>}

      <Form.Item label="名称" name="name">
        <Input aria-label="名称" />
      </Form.Item>
      {errors.name && <p role="alert">{errors.name}</p>}

      <Form.Item label="分类" name="categoryId">
        <Select
          aria-label="分类"
          options={categories.map((c) => ({ label: c.name, value: c.id }))}
          placeholder="请选择"
        />
      </Form.Item>
      {errors.categoryId && <p role="alert">{errors.categoryId}</p>}

      <Form.Item label="进价" name="costPrice">
        <InputNumber aria-label="进价" step={0.01} style={{ width: '100%' }} />
      </Form.Item>
      {errors.costPrice && <p role="alert">{errors.costPrice}</p>}

      <Form.Item label="售价" name="sellPrice">
        <InputNumber aria-label="售价" step={0.01} style={{ width: '100%' }} />
      </Form.Item>
      {errors.sellPrice && <p role="alert">{errors.sellPrice}</p>}

      <Form.Item label="初始库存" name="quantity">
        <InputNumber aria-label="初始库存" style={{ width: '100%' }} />
      </Form.Item>
      {errors.quantity && <p role="alert">{errors.quantity}</p>}

      <Form.Item label="单位" name="unit">
        <Input aria-label="单位" />
      </Form.Item>
      {errors.unit && <p role="alert">{errors.unit}</p>}

      <Form.Item label="供应商" name="supplierId">
        <Select
          aria-label="供应商"
          options={suppliers.map((s) => ({ label: s.name, value: s.id }))}
          placeholder="请选择"
        />
      </Form.Item>
      {errors.supplierId && <p role="alert">{errors.supplierId}</p>}

      <Form.Item label="最小库存" name="minStock">
        <InputNumber aria-label="最小库存" style={{ width: '100%' }} />
      </Form.Item>
      {errors.minStock && <p role="alert">{errors.minStock}</p>}

      <Form.Item label="最大库存" name="maxStock">
        <InputNumber aria-label="最大库存" style={{ width: '100%' }} />
      </Form.Item>
      {errors.maxStock && <p role="alert">{errors.maxStock}</p>}

      <Form.Item label="描述" name="description">
        <Input.TextArea aria-label="描述" rows={3} />
      </Form.Item>

      <Button
        type="primary"
        htmlType="submit"
        loading={submitting}
        disabled={submitting}
      >
        {submitting ? '保存中...' : '保存'}
      </Button>
    </Form>
  )
}
```

> 关键：`aria-label` 与字段中文标签一致 → 测试用 `getByLabel('编码')` 即可定位 input；`role="alert"` 包裹错误文案；按钮文案精确为"保存"或"保存中..."。
> Antd Form 自带的校验消息**不使用**（避免与 Zod 重复 → 文案不同步）；只用 `<p role="alert">`。

## 4. 各模块页面要点速查

### 4.1 M0 auth `/login`

- 表单字段：`用户名`、`密码`
- 提交按钮：`登录` / 提交中：`登录中...`
- 校验文案：`请输入用户名` / `请输入密码`
- 服务端 `INVALID_CREDENTIALS` 显示在表单上方 `role="alert"`：`用户名或密码错误`
- 登录成功后：跳 `redirect` 查询参数指向的页面（默认 `/dashboard`）

### 4.2 M1 categories

- 列表列：`名称`、`描述`、`创建时间`、`操作（编辑/删除）`
- 表单字段：`分类名称`（必填）、`描述`（可选）
- 删除前二次确认（同 §00 §7.4）

### 4.3 M2 suppliers

- 列表列：`供应商名称`、`联系电话`、`地址`、`操作`
- 表单字段：`供应商名称`（必填）、`联系电话`（可选）、`地址`（可选）

### 4.4 M3 products

- 见 §3.1 / §3.2
- 编辑页路由：`/products/[id]/edit`，编辑时 `code` 字段 disabled

### 4.5 M4 purchase-orders

- 列表列：`进货单号`、`商品名称`、`进货数量`、`进货日期`、`供应商`、`进货员`、`状态`、`操作`
- 状态显示：`草稿` / `已确认`（中文标签）
- 操作列：DRAFT 时显示 `编辑`、`删除`、`确认`；CONFIRMED 时显示 `查看`
- 表单字段：`商品`（select：显示 `code - name`）、`进货数量`、`进货单价`（默认填商品 costPrice）、`进货金额`（自动计算，禁用）、`供应商`、`进货员`、`备注`、`进货日期`
- 详情页 `/purchase-orders/[id]`：DRAFT 时按钮 `确认进货单` + `编辑` + `删除`；CONFIRMED 时按钮置空，显示 `已确认`、`确认时间`

### 4.6 M5 sales-orders

- 同 M4，把"供应商"换"客户"，"进货员"换"出货员"，"进货"换"出货"
- 表单字段：`商品`、`出货数量`、`出货单价`（默认填商品 sellPrice）、`出货金额`、`客户`、`出货员`、`备注`、`出货日期`
- 详情页：DRAFT 显示 `确认出货单`；CONFIRMED 时如果 `INSUFFICIENT_STOCK` 该如何处理？— **不会发生**：`confirm` 阶段才检查；DRAFT 单存在不影响 `quantity`，确认时再校验。

### 4.7 M6 stats

- `/dashboard`：4 张统计卡片（总商品数 / 总库存数 / 总库存价值 / 预警数）+ 周期统计表（今日/本周/本月 进货/出货 数量）
- `/stats/alerts`：表格列出预警商品（含 `alertType` 列：`低库存` / `超容量`）
- `/stats/trends`：选商品 + 日期范围 → 折线图（用 `@ant-design/plots` 或 `recharts`，项目选 `@ant-design/plots@^2`）
- `/stats/report`：按钮 `导出 Excel`，点击后调 `stats.report` 拉数据，前端用 `xlsx` 转 .xlsx 触发下载（文件名 `inventory-report-YYYYMMDD.xlsx`）

## 6. 错误展示约定

1. **字段级**（表单内）：在对应字段下方使用 `<p role="alert">{字面量}</p>`。文案见 `00-技术约束.md` §7.1。
2. **页面级**（如服务端错误）：在表单顶部使用 `<Alert role="alert" type="error" showIcon message={...} />`。文案见 §7.2。
3. **toast**（操作成功/批量结果）：`App.useApp().message.success(...)` / `.error(...)`。文案见 §7.3。
4. **二次确认弹窗**：使用 antd `Modal.confirm`；`okButtonProps={{ 'data-testid': 'modal-confirm' }}` 与 `cancelButtonProps={{ 'data-testid': 'modal-cancel' }}`。

> **绝不允许**让 antd Form 自动展示英文 zod issue（如 `Number must be greater than 0`）。所有面向用户文案必须**先 trip Zod，再翻译为字面量**。

## 7. Excel 导出（M6）

```ts
// src/components/stats/export-button.tsx
'use client'
import { Button } from 'antd'
import { trpc } from '@/lib/trpc-client'
import * as XLSX from 'xlsx'
import dayjs from 'dayjs'

export function ExportReportButton() {
  const utils = trpc.useUtils()
  return (
    <Button
      data-testid="report-export"
      onClick={async () => {
        const rows = await utils.stats.report.fetch()
        const ws = XLSX.utils.json_to_sheet(rows)
        const wb = XLSX.utils.book_new()
        XLSX.utils.book_append_sheet(wb, ws, '库存报表')
        XLSX.writeFile(wb, `inventory-report-${dayjs().format('YYYYMMDD')}.xlsx`)
      }}
    >
      导出 Excel
    </Button>
  )
}
```

> 在 package.json 添加 `xlsx@^0.20.0`（schema-agent 加）。
> 测试中：监听 `page.on('download')` 来断言文件名前缀。

## 8. 加载与空态

- 列表加载：antd Table 的 `loading` 属性
- 空态：使用 antd `<Empty>`，**必须**附 `data-testid="<module>-empty"` 才容易在测试中检测
- 提交中：按钮 disabled + 文案变成 `保存中...` / `登录中...` / `确认中...`

## 9. 客户端校验"零请求"约束（重要）

frontend-impl 必须保证：当客户端 Zod 校验失败时，**不发起任何 tRPC 请求**。这意味着：

- 表单提交前先 `safeParse`，失败则 `setErrors` 并 `return`，**不调** mutation
- E2E 会监听 `**/api/trpc/**` 请求计数，提交无效表单 → 计数应为 0

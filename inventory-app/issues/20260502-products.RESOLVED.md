# products: ProductForm 进价/售价 = 0 时校验文案偏离 spec

- 提交者：products-extra-tests-agent
- 日期：2026-05-02
- 模块：`products`
- 关联文档：`server/specs/products.md` §4.2、`server/00-技术约束.md` §7.1、`server/04-前端规范.md` §6

## 现象

`tests/component/products/product-form.test.tsx` 中以下两个 case 因实现偏离 spec 暂以 `it.skip` 跳过：

1. `ProductForm — 字段校验文案 > 进价 0 → 进价必须大于0`
2. `ProductForm — 字段校验文案 > 售价 0 → 售价必须大于0`

实际触发表单提交（`costPrice=0` 或 `sellPrice=0`）时，页面 `role="alert"` 的文本为：

```
Number must be greater than 0
```

而 spec（`00-技术约束.md` §7.1 与 `specs/products.md` §4.2）要求的字面量为：

```
进价必须大于0
售价必须大于0
```

诊断方式：在测试中 `screen.queryAllByRole('alert').map(el => el.textContent)` 读到 `[ 'Number must be greater than 0' ]`。

## 根因（推断，未读 src）

`src/components/products/product-form.tsx` 中 zod schema 推测为：

```ts
costPrice: z.number().positive().max(99999.99).refine(...)
sellPrice: z.number().positive().max(99999.99).refine(...)
```

未给 `.positive(...)` 提供中文 message 参数，导致回退到 zod 默认英文文案 `Number must be greater than 0`。

正确写法（与 `04-前端规范.md` §3.2 骨架一致）：

```ts
costPrice: z.number().positive('进价必须大于0').max(99999.99).refine(...)
sellPrice: z.number().positive('售价必须大于0').max(99999.99).refine(...)
```

## 影响

- `04-前端规范.md` §6 明确禁止把英文 zod issue 透到 UI；该实现违反该约束。
- 阻塞 component 测试 2 项（已 `it.skip`，待 impl 修复后回滚 skip）。
- e2e 5.x 中如有针对该文案的断言会同样失败。

## 期望修复

让 `ProductForm` 中：
- `costPrice` 的 `.positive()` 消息为 `'进价必须大于0'`；
- `sellPrice` 的 `.positive()` 消息为 `'售价必须大于0'`。

修复后请把 `it.skip` 改回 `it`，并重新运行：

```
pnpm tsc --noEmit
pnpm test:component:products
```

## 其他注意

`进价 10.123 → 进价最多保留2位小数` 当前 case **通过**，说明 `.refine(...)` 的中文 message 已就位；缺失只在 `.positive()` 一节。

## 解决（2026-05-03）
- 把 `moneyField` 的 `.positive()` / `.max()` 删除，改在字段级 `.refine` 提供中文文案
- 已 unskip 两条 component 测试

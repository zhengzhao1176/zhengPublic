# 库存管理系统 - 业务需求文档

## 1. 项目概述

### 1.1 项目名称
库存管理系统

### 1.2 项目目标
建立一个Web化的库存管理平台，用于企业对货物、物资进行实时追踪和管理，提升库存管理效率，降低库存成本。

### 1.3 适用范围
- 商品进出库管理
- 库存数量实时查询
- 库存预警提醒
- 库存统计分析

---

## 2. 功能需求

### 2.1 商品管理模块

#### 2.1.1 商品查询（Read）
- 显示所有商品列表
- 支持按商品名称、商品编码搜索
- 支持按分类、库存状态筛选
- 显示商品基本信息：编码、名称、分类、单价、当前库存量、单位、供应商等
- 支持分页展示，每页显示10-50条记录

#### 2.1.2 商品新增（Create）
- 填写商品基本信息表单：
  - 商品编码（必填，唯一）
  - 商品名称（必填）
  - 商品分类（必选）
  - 商品描述（可选）
  - 进价（必填）
  - 售价（必填）
  - 初始库存量（必填）
  - 计量单位（必选）
  - 供应商（必填）
  - 最小库存预警值（必填）
  - 最大库存容量（必填）
- 表单验证：数值类字段必须为正数，编码长度3-20字符
- 新增成功后显示成功提示，返回商品列表

#### 2.1.3 商品编辑（Update）
- 选择商品后编辑其信息
- 可修改字段：商品名称、分类、描述、进价、售价、计量单位、供应商、预警值、容量上限
- 编码、创建时间不可修改
- 修改前显示原始值，修改后进行验证确认
- 编辑成功后显示成功提示

#### 2.1.4 商品删除（Delete）
- 支持单个删除和批量删除
- 删除前进行二次确认提示
- 库存量大于0的商品不允许删除，提示用户先清空库存
- 删除成功后列表自动刷新

---

### 2.2 进货管理模块

#### 2.2.1 进货单查询
- 显示所有进货单列表
- 显示：进货单号、商品名称、进货数量、进货日期、供应商、进货员、备注
- 支持按进货单号、日期范围、供应商筛选
- 支持查看进货单详情

#### 2.2.2 进货单新增
- 创建进货单，选择商品并输入进货数量
- 自动计算进货金额（数量 × 进价）
- 输入供应商、备注信息
- 进货单提交后，系统自动增加对应商品的库存量
- 生成进货单号（系统自动生成，格式：RH + 日期 + 序列号）

#### 2.2.3 进货单编辑
- 未确认的进货单可以编辑
- 已确认的进货单仅可查看

#### 2.2.4 进货单删除
- 未确认的进货单可删除
- 删除时需二次确认

---

### 2.3 出货管理模块

#### 2.3.1 出货单查询
- 显示所有出货单列表
- 显示：出货单号、商品名称、出货数量、出货日期、客户、出货员、备注
- 支持按出货单号、日期范围、客户筛选
- 支持查看出货单详情

#### 2.3.2 出货单新增
- 创建出货单，选择商品并输入出货数量
- 检查库存是否充足，不足提示用户
- 自动计算出货金额（数量 × 售价）
- 输入客户、备注信息
- 出货单提交后，系统自动减少对应商品的库存量
- 生成出货单号（系统自动生成，格式：CH + 日期 + 序列号）

#### 2.3.3 出货单编辑
- 未确认的出货单可以编辑
- 已确认的出货单仅可查看

#### 2.3.4 出货单删除
- 未确认的出货单可删除
- 删除时需二次确认

---

### 2.4 库存统计模块

#### 2.4.1 库存总览
- 显示总商品数、总库存价值、库存预警数量
- 展示库存周期统计（日、周、月）

#### 2.4.2 库存预警
- 显示库存低于最小预警值的商品
- 显示库存超过最大容量的商品
- 支持按预警类型筛选（低库存、超容量）

#### 2.4.3 库存趋势
- 图表展示指定商品的库存变化趋势（可选日期范围）
- 图表展示各分类商品库存占比

#### 2.4.4 库存报表
- 支持导出库存统计报表（Excel格式）
- 报表包含：商品编码、名称、分类、当前库存、进价、库存价值等

## 4. 业务规则

### 4.1 库存规则
- 进货后库存数量 = 原库存 + 进货数量
- 出货后库存数量 = 原库存 - 出货数量
- 出货数量不能超过当前库存量
- 库存数量任何时刻不能为负数

### 4.2 删除规则
- 库存量>0的商品不能直接删除
- 进货单或出货单一旦确认，不能删除，仅能查看和作废

### 4.3 预警规则
- 库存低于最小预警值时，在库存预警模块显示
- 库存超过最大容量时，在库存预警模块显示
- 支持邮件或系统通知提醒（可选）

### 4.4 编码规则
- 商品编码必须唯一，长度3-20字符
- 进货单号：RH + YYYYMMDD + 4位序列号（如RH202404280001）
- 出货单号：CH + YYYYMMDD + 4位序列号（如CH202404280001）

## 5. 用户界面需求

### 5.1 页面布局
- 左侧菜单栏：导航功能模块
- 顶部头部：用户信息、登出按钮、系统时间
- 主体区域：各模块内容显示

### 5.2 核心页面
1. **仪表板（Dashboard）** - 库存总览、预警统计
2. **商品管理** - 商品列表、新增、编辑、删除
3. **进货管理** - 进货单列表、新增、编辑、删除
4. **出货管理** - 出货单列表、新增、编辑、删除
5. **库存统计** - 预警查看、趋势图表、报表导出
6. **分类管理** - 分类维护（可选）
7. **供应商管理** - 供应商维护（可选）

### 5.3 交互需求
- 所有列表支持搜索、筛选、分页
- 所有删除操作需二次确认
- 表单验证实时进行，错误提示清晰
- 操作成功/失败都需要反馈提示
- 数据修改后显示操作者和操作时间

## 6. 技术要求

### 6.1 技术栈建议
- **前端**：React / Vue / Angular
- **后端**：Node.js / Python / Java / Go
- **数据库**：MySQL / PostgreSQL
- **其他**：RESTful API / GraphQL

### 6.2 性能要求
- 商品列表加载时间 < 2秒
- 单次操作响应时间 < 1秒
- 支持至少1万条商品记录

### 6.3 安全需求
- 用户身份认证（登录）
- 操作权限控制（不同角色不同权限）
- 数据加密存储（敏感信息）
- API请求验证和限流

### 6.4 浏览器兼容性
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+
